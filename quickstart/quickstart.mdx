## 3. Add your Autumn secret key

Create an [Autumn Secret API key](https://app.useautumn.com/dev) from the dashboard, and add it to your `.env` file.

```
AUTUMN_SECRET_KEY=am_sk_1234567890
```

## 4. Mount the Autumn handler

This creates routes in the `/api/autumn/*` path, used by Autumn's React [hooks](/docs/hooks).

The handler takes in an identify function that returns a `customerId`. This is typically your internal user ID or organization ID from your auth provider. These examples use better-auth.

<CodeGroup>

```typescript Next.js
// app/api/autumn/[...all]/route.ts

import { autumnHandler } from "autumn-js/next";
import { auth } from "@/lib/auth";

export const { GET, POST } = autumnHandler({
  identify: async (request) => {
    const session = await auth.api.getSession({
      headers: request.headers,
    });

    return {
      customerId: session?.user.id,
      customerData: {
        name: session?.user.name,
        email: session?.user.email,
      },
    };
  },
});
```

```typescript Remix
// app/routes/api.autumn.$.ts

import { autumnHandler } from "autumn-js/remix";
import { auth } from "../lib/auth.server";

export const { loader, action } = autumnHandler({
  identify: async (args) => {
    const session = await auth.api.getSession({
      headers: args.request.headers,
    });

      customerId: session?.user.id,
      customerData: {
        name: session?.user.name,
        email: session?.user.email,
      },
    };
  },
});
```

## 4. Set up `<AutumnProvider />`

Client side, wrap your application with the `<AutumnProvider>` component, and pass in your backend URL.

<Note>Backend URL is typically passed in as an environment variable.</Note>

```jsx
import { AutumnProvider } from "autumn-js/react";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode,
}) {
  return (
    <html>
      <body>
        <AutumnProvider backendUrl="http://localhost:8000">
          {children}
        </AutumnProvider>
      </body>
    </html>
  );
}
```

<Info>
  `AutumnProvider` can also take in a `getBearerToken` function, which should be used
  for auth frameworks with separate backend and frontend, eg using Clerk with
  Express.

</Info>

## 5. Integrate Autumn

There are 3 key functions you need to integrate the Pro plan (and any other pricing model) into your app:

<Steps>
  <Step>
    When the customer clicks the "Upgrade to Pro" button, we call `attach()` to
    redirect them to a Stripe checkout page.
    <Warning>
      Make sure you've pasted in your [Stripe test secret
      key](https://dashboard.stripe.com/test/apikeys) `sk_test_...` in the
      [Autumn dashboard](https://app.useautumn.com/integrations/stripe).
    </Warning>
  </Step>
  <Step>
    When the customer wants to use a chatbot message, we call `check()` to check
    if they have remaining messages and are allowed to send one.
  </Step>
  <Step>If they are, we call `track()` to send a usage event to Autumn.</Step>
</Steps>

<Note>
From the frontend, you can use Autumn's React hooks to check feature permissions, handle billing flows and display customer billing data.

Autumn's backend SDK can be used to also check feature permissions, and track feature usage.

</Note>

<CodeGroup>

```jsx Client
// app/chatbot/page.tsx

import { useAutumn, useCustomer } from "autumn-js/react";

export default function BillingPage() {
  const { attach, check } = useAutumn();
  const { customer, refetch } = useCustomer();

  return (
    <>
      <h1>Welcome {customer?.name}</h1>
      <button onClick={async () => await attach({ productId: "pro" })}>
        Purchase
      </button>
      <button
        onClick={async () => {
          let { data } = await check({ featureId: "messages" });
          if (!data?.allowed) {
            alert("You're out of messages");
          }
        }}
      >
        Send
      </button>
    </>
  );
}
```

```typescript Server
// app/chatbot/page.tsx
import { Autumn } from "autumn-js";

const autumn = new Autumn({ secretKey: "am_sk_..." });

autumn.track({
  customer_id: "user_123",
  feature_id: "messages",
});
```

</CodeGroup>
